# 自改进 Agent 学习文章系列

这个目录用于沉淀中文学习文章。文章不是单纯概念综述，而是围绕本仓库的可运行实验、结构化 trace、评估结果和反思记录展开。

## 写作原则

每篇文章尽量绑定一个可观察对象：

- 一段最小代码。
- 一个固定任务集。
- 一份运行 trace。
- 一个 evaluator 或 rubric。
- 一次 baseline 与改进版本的对比。
- 一条从失败案例提炼出的 memory / SOP / workflow 规则。

如果某篇文章暂时没有实验支撑，需要明确标记为来源整理或作者推理，后续再补实验验证。

## 文章模板

```text
# 标题

## 问题

这篇文章要回答什么问题？这个问题为什么值得通过实验观察？

## Source Status

- Official facts:
- Paper claims:
- Community implementation:
- Author inference:
- Local observation:

## 机制解释

解释相关机制，不把推理当成官方事实。

## 最小实验

说明实验输入、运行方式、关键代码和可复现边界。

## 实验观察

记录 trace、评分、失败样例或对比结果。

## 工程启发

把观察转成可以复用的规则、SOP、prompt 约束或后续实验问题。
```

## 与代码实验的关系

文章和实验交替推进：

1. 先实现一个足够小的可运行切片。
2. 保存 trace 和结果。
3. 基于结果写文章。
4. 从文章中提炼下一轮实验问题。

这样文章不会脱离实践，代码也不会只剩工程骨架。
